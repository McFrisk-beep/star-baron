# Cosmocrat — Graphics Spec & Art Drop-in Guide

Everything visual in Cosmocrat is a PNG loaded by **path + filename**. To put
real art in the game you **replace a placeholder PNG with your own, keeping the
same path, filename, and (ideally) dimensions** — no code changes needed. The
engine already falls back to a tinted box / drawn shape if a file is missing, so
you can ship a partial art set and fill in the rest over time.

> Placeholders are generated by `tools/gen_placeholders.py` (`pip install
> pillow`, then `python3 gen_placeholders.py` from `/tools`). It overwrites the
> placeholder set; it never touches files you've hand-replaced elsewhere.

---

## 1. Golden rules

1. **Keep the filename and folder.** The path *is* the API (see `ASSET` in
   `js/data.js`). `assets/planets/lava.png` will always be the lava planet.
2. **Keep the dimensions** (or the same aspect ratio). Sizes below are what the
   game draws at; off-size art is scaled and may look soft or stretched.
3. **PNG, RGBA.** Sprites use **transparency** (alpha) so they sit on starfields
   and nebulae. Full-frame backdrops (nebulae, broadcast frames) can be opaque.
4. **Design at the listed size or an exact 2× multiple** for crispness on HiDPI.
5. **Center the subject** with a little padding; the engine anchors most sprites
   at their center. Ships are the exception (nose points +X / right — see §6).
6. **Mind contrast.** Sprites render on dark space; keep a subtle rim/edge so
   they read against both black and bright nebulae.

## 2. Asset manifest (every image the game loads)

| Category | Path pattern | Size (px) | Transparent | Count | Drawn via | Used in |
|---|---|---|---|---|---|---|
| Alien portraits | `assets/portraits/alien_NN.png` | 96×96 | ✅ | 12 (`portraitCount`) | `<img>` | chat feed, NPCs |
| Commodity icons | `assets/commodities/<id>.png` | 64×64 | ✅ | 12 | `<img>` | exchange table |
| Fleet ships | `assets/ships/<sprite>.png` | 96×64 | ✅ | 4 | `<img>` | fleet panel |
| Broadcast frames | `assets/broadcast/<name>.png` | 384×216 (16:9) | optional | 5 | `<img>` | TV / news screen |
| **Stars (map nodes)** | `assets/stars/<type>.png` | 48×48 | ✅ | 7 | SVG `<image>` | galaxy view |
| **Planets** | `assets/planets/<type>.png` | 64×64 | ✅ | 9 | `<canvas>` + `<img>` | system view + info |
| **Race ships (tiny)** | `assets/raceships/<race>.png` | 32×20 | ✅ | 6 | `<canvas>` | system view traffic |
| **Space stations** | `assets/stations/<race>.png` | 64×64 | ✅ | 6 | `<canvas>` | system view |
| **Nebula backdrops** | `assets/nebula/<name>.png` | 640×360 (16:9) | opaque ok | 6 | `<canvas>` | system view background |
| **Asteroids** | `assets/space/asteroids.png` | 128×128 | ✅ | 1 | `<canvas>` | asteroid-belt rings |

Bold rows are the new star-map art. The asset path helpers live in
`js/data.js`:

```js
ASSET.portrait(i)     // assets/portraits/alien_07.png
ASSET.commodity(id)   // assets/commodities/helium3.png
ASSET.ship(sprite)    // assets/ships/freighter.png
ASSET.broadcast(name) // assets/broadcast/news.png
ASSET.star(type)      // assets/stars/blue.png
ASSET.planet(type)    // assets/planets/gas_giant.png
ASSET.raceship(race)  // assets/raceships/krell.png
ASSET.station(race)   // assets/stations/glorthi.png
ASSET.nebula(name)    // assets/nebula/purple.png
ASSET.asteroids()     // assets/space/asteroids.png
```

## 3. The races (palette + voice)

Six playable-flavor races. Each has a signature color used for its ship,
station, and sector accents. Match these hues so a system "reads" as its race at
a glance. Defined in `RACES` (`js/data.js`).

| Race id | Name | Color | Vibe to aim for |
|---|---|---|---|
| `voidkin`  | Voidkin  | `#7b8cff` indigo | sleek, calm, diplomatic core-world tech |
| `glorthi`  | Glorthi  | `#3ad6a0` jade | organic, bulbous, gas-harvester biotech |
| `aurelian` | Aurelian | `#ffc24b` gold | regal, ornate, agrarian gilded barges |
| `krell`    | Krell    | `#ff5d73` red | aggressive, angular, industrial war-forge |
| `mechanim` | Mechanim | `#9aa9c8` steel | blocky, modular, mining drones & rigs |
| `syndics`  | Syndics  | `#a078ff` violet | slick, luxe, shady smuggler couture |

You provide **one tiny ship + one station per race**. Keep silhouettes
distinct between races (a Krell ship should be unmistakable vs a Voidkin one).

## 4. Stars (galaxy-map nodes) — `assets/stars/<type>.png`

- **Size:** 48×48, transparent. Drawn in the galaxy SVG at **16px** for normal
  systems and **26px** for sector capitals, centered on the node.
- **Read at tiny size:** these are points on a map. Strong central core +
  soft corona; avoid fine detail that vanishes at 16px.
- **Types (filenames):** `yellow`, `blue`, `red`, `white`, `orange`,
  `neutron`, `binary`. (`binary` = two cores.)
- Color should match the type name; the surrounding node ring is drawn by the
  engine (green = rising, red = falling, amber = local event), so don't bake a
  ring into the art.

Add a star type: append to `STAR_TYPES` in `js/data.js` and drop
`assets/stars/<newtype>.png`.

---

## 5. Planets — `assets/planets/<type>.png`

- **Size:** 64×64, transparent, centered sphere with a few px of padding.
- **Drawn:** in the system-view canvas (16–28px radius, orbiting) **and** as a
  40×40 thumbnail in the info panel's planet list. Make it legible at both.
- **Lighting:** light from the upper-left (highlight top-left, terminator
  bottom-right) to match the in-scene star glow.
- **Types (filenames):** `rocky`, `terran`, `ocean`, `ice`, `lava`,
  `gas_giant`, `barren`, `ringed`, `toxic`.
  - `gas_giant` / `ringed` are drawn larger in-scene — give them presence.
  - `ringed` rings may extend slightly past the 64×64 box edges; keep the ring
    within the canvas so it isn't clipped (the placeholder fits it inside).

Add a planet type: append to `PLANET_TYPES` in `js/data.js` and drop
`assets/planets/<newtype>.png`.

---

## 6. Race ships (system traffic) — `assets/raceships/<race>.png`

- **Size:** 32×20, transparent. Drawn at **20×12** in the canvas, **rotated to
  its heading**.
- **Orientation: nose points RIGHT (+X).** The engine rotates the sprite to its
  flight direction, so draw it facing right at rest. Put the **engine/exhaust on
  the LEFT** (the placeholder adds a small blue thruster glow there).
- One sprite per race (`voidkin`, `glorthi`, `aurelian`, `krell`, `mechanim`,
  `syndics`). These are the "teeny-tiny ships" flitting between planets and the
  station — silhouette + race color matter far more than detail at this size.

---

## 7. Space stations — `assets/stations/<race>.png`

- **Size:** 64×64, transparent. Drawn at **32×32**, centered, on its own short
  orbit near the star.
- One per race; lean into race identity (Aurelian = ornate ring; Mechanim =
  blocky industrial; Syndics = sleek dark hub).
- A small lit core/beacon reads well as "this is alive."

---

## 8. Nebula backdrops — `assets/nebula/<name>.png`

- **Size:** 640×360 (16:9). Can be **opaque** (it's the scene background).
- **Drawn:** stretched to **fill** the system-view scene (it will be scaled and
  may be cropped on extreme aspect ratios — keep important detail centered).
- **Keep it dark/low-contrast.** Planets, ships, and the star render on top; a
  busy or bright nebula hurts readability. Aim for deep tones + a faint
  starfield. The placeholder bakes in stars; you can too.
- **Names (filenames):** `blue`, `red`, `green`, `purple`, `gold`, `void`.
  Each sector picks one (see `SECTORS[].nebula` in `js/data.js`):
  Core→`void`, Belt→`blue`, Tide→`green`, Green Expanse→`gold`,
  Forge→`red`, Sprawl→`purple`.

Add a nebula: drop `assets/nebula/<name>.png` and reference it from a sector's
`nebula` field.

---

## 9. Asteroids — `assets/space/asteroids.png`

- **Size:** 128×128, transparent. A scattered cluster of rocks on transparency.
- **Drawn:** stamped repeatedly at 32×32 around a belt ring for systems flagged
  `asteroidBelt` (mineral-rich systems and ~40% of generated ones).
- Keep rocks small and varied; the engine creates the "belt" by tiling, so a
  single dense-ish cluster tile works best.

---

## 10. The existing (non-map) art, for completeness

- **Portraits** `assets/portraits/alien_00..11.png` (96×96, transparent):
  chat avatars. NPCs are pinned to specific indices in `flavor.js` (`NPCS[].portrait`).
- **Commodity icons** `assets/commodities/<id>.png` (64×64): one per commodity
  id in `COMMODITIES` (`iron_ore`, `helium3`, `antimatter`, …).
- **Fleet ships** `assets/ships/<sprite>.png` (96×64): `shuttle`, `hauler`,
  `freighter`, `leviathan` — the ships you own and dispatch (distinct from the
  ambient `raceships`).
- **Broadcast frames** `assets/broadcast/<name>.png` (384×216): `news`,
  `tv_drama`, `tv_ads`, `tv_weather`, `static` — the TV/news screen backgrounds.

## 11. Adding new content (art + data together)

The galaxy is generated from data, so new art usually means appending to a list
in `js/data.js` (or `js/flavor.js`) and dropping the matching PNG:

- **New race:** add to `RACES` (id, name, color) + `assets/raceships/<id>.png`
  + `assets/stations/<id>.png`.
- **New star / planet type:** add to `STAR_TYPES` / `PLANET_TYPES` +
  the matching PNG.
- **New nebula:** drop the PNG, point a sector's `nebula` at it.
- **New sector / capital:** add a `SECTORS` entry (and a `SYSTEMS` capital it
  points at). Sector count, themes, and specialties live there.

After changing the data lists, re-run `gen_placeholders.py` to get fresh
placeholders for any new ids, then replace them with real art when ready.

---

## 12. Art-direction notes (open)

The current placeholders are **deliberately neutral** (flat, geometric, single
light source) so they don't commit you to a style. When you pick a direction —
painterly sci-fi, clean vector/flat, pixel-art, retro-terminal, etc. — keep it
**consistent across a whole category first** (e.g. all planets, then all ships)
so the game never looks half-reskinned. The race palette in §3 is the one thing
worth preserving across any style, because gameplay reads color as identity.

> Tip: because the engine falls back gracefully, you can drop in one finished
> category at a time (say, the 9 planets) and immediately see it in-game via the
> Star Map without touching code.
